Descripción: Este archivo enumera, línea por línea, especificaciones y notas del proyecto HydroLeaf.
Alcance: Solo contenido del proyecto HydroLeaf (plataforma web para gestión de cultivos hidropónicos).
Formato: Una idea por línea; útil para diffs, revisiones rápidas y pruebas de commits.
Autoría: Contribución documental al repositorio HydroLeaf.
== Resumen General ==
HydroLeaf: plataforma web para automatizar y monitorear sistemas hidropónicos.
Objetivo: optimizar agua y nutrientes con decisiones basadas en datos.
Frontend: React con Vite; enrutamiento con React Router.
Backend: Node.js + Express; API REST.
Comunicación IoT: MQTT (simulación en etapas iniciales).
Base de datos: prevista (MongoDB o MySQL) para históricos.
UI: diseño responsivo móvil/escritorio; dashboard, zonas, riego, alertas.
Seguridad: CORS; autenticación prevista para sprints posteriores.
Despliegue: local en desarrollo; futuro en nube (Render/Vercel/Heroku).
Licencia: académica/educativa según el repositorio.
== Objetivos ==
Reducir desperdicio de agua vía umbrales de riego por zona.
Monitorear humedad y temperatura en tiempo real (simulado).
Permitir control manual/automatizado del riego.
Registrar eventos y estados para trazabilidad.
Ofrecer experiencia simple y clara para usuarios no técnicos.
== Arquitectura ==
Arquitectura cliente-servidor con API REST.
Capas: presentación (React), lógica (Express), datos (DB futura).
Patrón propuesto: controladores/servicios en backend.
Estrategia MQTT: topicos /hydroleaf/sensors y /hydroleaf/actuators.
Gestión de estados en frontend: hooks y contexto cuando aplique.
== Historias de Usuario (HU) ==
HU01: Visualizar humedad y temperatura por zona.
HU02: Activar riego manual por zona.
HU03: Registrar incidencias y eventos.
HU04: Configurar umbrales de humedad.
HU05: Visualizar alertas y notificaciones locales.
HU06: Consultar historial de mediciones (simulado).
== Endpoints previstos ==
GET /api/sensors → listado de sensores y último valor simulado.
GET /api/sensors/:zoneId → detalle de una zona.
POST /api/irrigation/start → iniciar riego en zona.
POST /api/irrigation/stop → detener riego en zona.
GET /api/history → lecturas históricas (cuando exista BD).
== Pantallas UI ==
Dashboard: tarjetas de humedad/temperatura y gráfico lineal.
Control de Riego: botones activar/detener y estado actual.
Gestión de Zonas: alta/baja/edición (futuro).
Alertas: lista de notificaciones locales.
Configuración: umbrales por zona y preferencias.
Historial: tabla de eventos y medidas (futuro).
